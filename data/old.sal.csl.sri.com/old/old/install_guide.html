A Step by Step Installation Guide


1. Installing the SAL Parser

Downloading the SAL Parser

Download - free of charge for non-commercial use under the FM license Please fill in this form and mail or fax to the address on the front cover. You will receive a login to download any SAL component when your license has been processed.

Configuring and Installing the SAL Parser

The file INSTALL inside the installation package contains instructions to install the SAL Parser.
  • Unpack the sources: 
     tar xvfz sal2xml-1.0.tar.gz
  • Change to the toplevel directory:
     cd sal2xml-1.0
  • Configure the SAL parser: 
     ./configure
    It is necessary to have a JAVA virtual machine to execute the SAL parser. If you do not have one, you can download one from http://www.javasoft.com. The root of the default installation directory is /usr/local, if you do not have super user privileges, you can install the SAL parser in your home directory. For instance, to install it at ~/code, you should use: 
     ./configure --prefix=~/code
    If you want to force a specific JAVA virtural machine, you can use the following command line: 
     ./configure JAVA=[your virtual machine]
    Example: 
     ./configure JAVA=java
  • Installing the SAL parser: 
     make install

2. Installing Bigloo (Scheme compiler)

Bigloo is a Scheme implementation developed by Manuel Serrano. The Bigloo homepage is http://www-sop.inria.fr/mimosa/fp/Bigloo/

Downloading Bigloo

You can download Bigloo from our website or the Bigloo official website.
  • Local copy
  • Bigloo at INRIA

Building Bigloo

The file INSTALL inside the installation package contains instructions to install Bigloo.
  • Unpack the sources: 
     tar -xvzf bigloo2.5a.tar.gz
  • Change to the toplevel directory: 
     cd bigloo2.5a
  • Configure Bigloo: 
     ./configure --jvm=no
    The --jvm=no disables the JAVA support in Bigloo.
    The root of the default installation directory is /usr/local, if you do not have super user privileges, you can install Bigloo in your home directory. For instance, to install it at ~/code, you should use: 
     ./configure  --jvm=no --prefix=~/code
  • Compiling Bigloo: 
     make
  • Installing Bigloo: 
     make install
  • Final configurations:

    On some architecture you will be needing to tell the loader where to find the SALenv and Bigloo shared libraries. This can be done two ways:
    • setting the shell LD_LIBRARY_PATH variable.
    • updating the `/etc/ld.so.conf' file (read by ldconfig man page).
    Otherwise, you will probably receive the following error message: 
     salenv: error in loading shared libraries: libsalcore.so: cannot open shared object file: No such file or directory
    SALenv libraries are installed in the Bigloo shared libraries directory. If you installed Bigloo in `usr/local', then the libraries are located at: 
     /usr/local/lib/bigloo/2.5a
    In this case, you can fix the problem by adding the following command to your `.bashrc' file: 
     export LD_LIBRARY_PATH=/usr/local/lib/bigloo/2.5a:$LD_LIBRARY_PATH

3. Installing GMP

GMP is the GNU multi precision library. This library is used by SALenv. The recent versions of Red Hat are coming with GMP, so you can try to jump this step. If you do not have the library, the SALenv installation script will generate an error message.

Downloading GMP

You can download GMP from our website or the GMP official website.
  • Local copy
  • Official web site

Building GMP

The file INSTALL inside the installation package contains instructions to install GMP. The installation is similar to the previous ones.
